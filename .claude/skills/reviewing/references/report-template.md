# Report Template

Structured template for review reports saved to `.claude/.reviews/`.

## Full Report Template

```markdown
# Code Review Report

**Date**: {YYYY-MM-DD HH:MM}
**Scope**: {scope_description}
**Type**: {staged | pr | commit | branch | files | working}
**Command**: `{git_command}`

---

## Summary

{2-3 sentence overall assessment of the changes reviewed}

### Statistics

| Category | Count |
| -------- | ----- |
| Critical | {n}   |
| Warning  | {n}   |
| Info     | {n}   |

### Verdict

**{APPROVE | PASS WITH WARNINGS | REQUEST CHANGES}**

{Brief reason for the verdict}

---

## Checklist

| Category       | Status | Notes                    |
| -------------- | ------ | ------------------------ |
| Functionality  | {x/o}  | {brief note if issues}   |
| Readability    | {x/o}  | {brief note if issues}   |
| Security       | {x/o}  | {brief note if issues}   |
| Performance    | {x/o}  | {brief note if issues}   |
| Testing        | {x/o}  | {brief note if issues}   |
| Documentation  | {x/o}  | {brief note if issues}   |
| Architecture   | {x/o}  | {brief note if issues}   |
| Error Handling | {x/o}  | {brief note if issues}   |

Legend: x = passed, o = issues found

---

## Findings

### Critical Issues (Must Fix)

{If no critical issues: "No critical issues found."}

#### `issue (blocking):` {file_path}:{line_number}

**Issue**: {Clear description of the problem}

**Impact**: {Why this is critical - security risk, bug, breaking change, etc.}

**Category**: {Security | Functionality | Performance | etc.}

**Fix**:
```{language}
{Suggested code fix or approach}
```

---

### Warnings (Should Fix)

{If no warnings: "No warnings found."}

#### `issue (non-blocking):` {file_path}:{line_number}

**Issue**: {Description of the problem}

**Impact**: {Potential problems this could cause}

**Category**: {Category}

**Fix**:
```{language}
{Suggested fix}
```

---

### Suggestions

{If no suggestions: "No additional suggestions."}

#### `suggestion:` {file_path}:{line_number}

{Improvement recommendation}

---

### Nitpicks

{Minor style or preference items}

#### `nitpick:` {file_path}:{line_number}

{Minor observation}

---

### Questions

{Areas needing clarification}

#### `question:` {file_path}:{line_number}

{Question about the code}

---

### Praise

{Good code patterns or practices noticed}

#### `praise:` {file_path}:{line_number}

{What was done well}

---

## Dispatcher Details

| Field     | Value                     |
| --------- | ------------------------- |
| Agent     | reviewer                  |
| Mode      | {single | parallel}       |
| Instances | {count}                   |
| Strategy  | {by_criteria | by_file | by_change} |

### Instance Inputs

{For each instance:}

**Instance {n}**
- Focus: {what this instance focused on}
- Scope: {specific scope/query}

---

## Files Reviewed

| File                  | Lines Changed | Issues Found |
| --------------------- | ------------- | ------------ |
| `{path/to/file1.ts}`  | +{n}/-{n}     | {count}      |
| `{path/to/file2.ts}`  | +{n}/-{n}     | {count}      |

---

## Review History

| Date       | Scope   | Verdict          | Critical | Warnings |
| ---------- | ------- | ---------------- | -------- | -------- |
| {today}    | {scope} | {verdict}        | {n}      | {n}      |

{If previous reviews exist for same scope, add comparison:}

### Comparison with Previous Review

- **Previous**: {date} - {verdict}
- **Current**: {date} - {verdict}
- **Improvement**: {description of changes}

---

_Generated by /review workflow_
_Report: .claude/.reviews/{filename}_
```

## Lite Report Template (Future Use)

For quick reviews or small changes:

```markdown
# Quick Review

**Date**: {YYYY-MM-DD}
**Scope**: {scope_description}

## Summary

{Brief assessment}

## Issues

| Severity | File | Line | Issue |
| -------- | ---- | ---- | ----- |
| {level}  | {f}  | {l}  | {i}   |

## Verdict

**{verdict}**

---

_Quick review by /review workflow_
```

## Variable Reference

| Variable              | Description                          |
| --------------------- | ------------------------------------ |
| `{YYYY-MM-DD HH:MM}`  | Full timestamp                       |
| `{scope_description}` | Human-readable scope description     |
| `{git_command}`       | Command used to get changes          |
| `{n}`                 | Count of issues/lines/etc.           |
| `{x/o}`               | Pass (x) or issues found (o)         |
| `{verdict}`           | APPROVE/PASS WITH WARNINGS/REQUEST CHANGES |
| `{file_path}`         | Path to file with issue              |
| `{line_number}`       | Line number of issue                 |
| `{language}`          | Code language for syntax highlighting |
